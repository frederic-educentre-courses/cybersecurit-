Cyber Sécurité

Qu'est-ce que la cyber sécurité

La cyber sécurité est l'ensemble des mesures et des pratiques visant à protéger les systèmes informatiques et les réseaux contre les menaces et les attaques.

Sanctions

En cas de cyber attaque, les sanctions peuvent être sévères, allant de l'amende à la prison en passant par des peines de travail d'intérêt général.

Lien vers les sanctions

Red Team et Blue Team

  • Le Red Team simule des attaques pour tester la résilience d'un système.
  • Le Blue Team protège le système contre ces attaques.

Liste d'attaques

  • Injection SQL
  • XSS (Cross-Site Scripting)
  • CSRF (Cross-Site Request Forgery)
  • File upload & command injection
  • Directory traversal
  • Attaques par force brute
  • Attaques DDoS

Injection SQL

L'injection SQL est une vulnérabilité de sécurité qui permet à un attaquant de manipuler les requêtes SQL d'une application web.

Red Team

Pour réaliser une attaque d'injection SQL, un attaquant pourrait utiliser des techniques comme l'injection de code malveillant dans les paramètres d'entrée.

Exemple :

SELECT * FROM users WHERE username = 'admin' OR '1'='1';

Blue Team

Pour prévenir les attaques d'injection SQL, le Blue Team peut utiliser des requêtes préparées et valider les entrées utilisateur.

<?php
$stmt = $pdo->prepare('SELECT * FROM users WHERE username = :username');
$stmt->execute(['username' => $username]);
$users = $stmt->fetchAll();
?>

XSS (Cross-Site Scripting)

XSS (Cross-Site Scripting) est une vulnérabilité de sécurité qui permet à un attaquant de injecter du code malveillant dans les pages web visitées par les utilisateurs.

Red Team

Pour réaliser une attaque XSS, un attaquant pourrait injecter du code malveillant dans les champs de saisie des utilisateurs.

Exemple :

<script>alert('XSS Attack!');</script>

En injectant ce code dans un champ de commentaire, l'attaquant pourrait faire apparaître une alerte sur la page de tous les utilisateurs qui consultent ce commentaire.

Blue Team

Pour prévenir les attaques XSS, le Blue Team peut utiliser des fonctions d'échappement lors de l'affichage des données utilisateur dans le HTML.


<section id="comment-section">
	...
	<?php
		echo htmlspecialchars($comment, ENT_QUOTES, 'UTF-8')
	?>
</section>

CSRF (Cross-Site Request Forgery)

Le CSRF (Cross-Site Request Forgery) est une vulnérabilité de sécurité qui permet à un attaquant de faire des requêtes non autorisées au nom d'un utilisateur authentifié.

Red Team

Un attaquant peut exploiter le CSRF en envoyant une requête malveillante à une application web en utilisant les cookies de session de l'utilisateur authentifié.

Exemple :

<form action="https://example.com/delete-account/1" method="POST">
	<input type="submit" value="Supprimer le compte" />
</form>

En envoyant ce formulaire à un utilisateur authentifié, l'attaquant pourrait supprimer le compte de cet utilisateur sans son consentement.

Blue Team

Pour prévenir les attaques CSRF, le Blue Team peut utiliser des tokens CSRF pour vérifier que les requêtes proviennent de sources légitimes.

<?php
session_start();
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
	if (!isset($_POST['csrf_token']) || $_POST['csrf_token'] !== $_SESSION['csrf_token']) {
		// Requête non autorisée
		header('HTTP/1.1 403 Forbidden');
		exit;
	}
?>

Blue Team (suite)

Le token est passé dans le formulaire :

<form action="https://example.com/delete-account/1" method="POST">
							<input type="hidden" name="csrf_token" value="<?php echo $_SESSION['csrf_token']; ?>" />
							<input type="submit" value="Supprimer le compte" />
						</form>

En vérifiant le token CSRF, l'application peut s'assurer que la requête est légitime et empêcher les attaques CSRF.

File upload & command injection

Le file upload & command injection est une vulnérabilité de sécurité qui permet à un attaquant de télécharger des fichiers malveillants ou d'exécuter des commandes sur le serveur.

Red Team

Un attaquant peut exploiter cette vulnérabilité en téléchargeant un fichier malveillant ou en injectant des commandes dans les champs de saisie.

Exemple :

Dans un formulaire permettant le téléchargement de fichiers, un attaquant pourrait uploader un fichier PHP malveillant et l'exécuter sur le serveur.

Red Team (suite)

Exemple de fichier PHP permettant l'exécution de commandes :

<?php
    if (isset($_GET['cmd'])) {
        system($_GET['cmd']);
    }
?>

En accédant à ce fichier avec une URL comme https://example.com/uploaded-file.php?cmd=ls, l'attaquant pourrait exécuter la commande "ls" sur le serveur et obtenir la liste des fichiers.

Blue Team

Pour prévenir les attaques par file upload & command injection, le Blue Team peut :

  • Valider le type de fichier avant le téléchargement
  • Renommer les fichiers téléchargés
  • Stocker les fichiers dans un répertoire non accessible via le web
  • Utiliser des méthodes de validation strictes pour les entrées utilisateur

Blue Team (suite)

Voici un exemple de code sécurisé pour le téléchargement de fichiers :

<?php
if (isset($_FILES['file'])) {
	$file = $_FILES['file'];
	// Valider le type de fichier
	$allowedTypes = ['image/jpeg', 'image/png'];
	if (in_array($file['type'], $allowedTypes)) {
		// Renommer le fichier
		$filename = uniqid() . '-' . basename($file['name']);
		// Stocker le fichier dans un répertoire sécurisé
		move_uploaded_file($file['tmp_name'], '/secure/uploads/' . $filename);
	} else {
		// Type de fichier non autorisé
		header('HTTP/1.1 400 Bad Request');
		exit;
	}
?>

Directory traversal

Le directory traversal est une vulnérabilité de sécurité qui permet à un attaquant d'accéder à des fichiers et des répertoires en dehors de la racine du serveur.

Red Team

Un attaquant peut accéder à des fichiers sensibles en utilisant des séquences de caractères comme "../" dans l'url.

Exemple : https://example.com/.env

Blue Team

Pour prévenir les attaques de directory traversal, le Blue Team peut configurer le serveur pour restreindre l'accès aux fichiers.

Attaque par brute force

Une attaque par brute force est une méthode d'attaque qui consiste à essayer toutes les combinaisons possibles de mots de passe ou de clés pour accéder à un système.

Red Team

Pour réaliser une attaque par brute force, un attaquant peut utiliser des outils automatisés pour essayer plusieurs combinaisons de mots de passe.

Blue Team

Pour prévenir les attaques par brute force, le Blue Team peut mettre en place des mesures de sécurité telles que :

  • Limiter le nombre de tentatives de connexion
  • Utiliser des mots de passe forts et uniques
  • Mettre en place une authentification à deux facteurs

DDoS

Un attaquant peut utiliser des techniques de DDoS pour surcharger les ressources d'un serveur et empêcher les utilisateurs légitimes d'accéder au service.

Red Team

Pour réaliser une attaque DDoS, un attaquant peut utiliser un botnet pour envoyer un grand nombre de requêtes à un serveur cible.

Blue Team

Pour prévenir les attaques DDoS, le Blue Team peut mettre en place des mesures de sécurité telles que :

  • Utiliser un service de protection DDoS
  • Configurer le serveur pour limiter le nombre de requêtes simultanées
  • Mettre en place un système de détection et de blocage des attaques DDoS
